1. Создаем структуру каталогов
mkdir -p ~/Frolov/1/2
mkdir -p ~/Frolov/1/3
mkdir -p ~/Frolov/4

2.Копирование файла group
cp /etc/group ~/Frolov/1/group
cp /etc/group ~/Frolov/1/2/group
cp /etc/group ~/Frolov/1/3/group
cp /etc/group ~/Frolov/4/group

3. Сведения о командах с помощью file
file /bin/ls
file /bin/cp
file /dev/null
file /dev/zero

4.Команда ls -l /dev
ls -l /dev | head -20 
ls - команда для вывода списка файлов и каталогов

-l - ключ для подробного вывода (long format)

/dev - каталог устройств в Linux
| - Перенаправляет вывод одной команды на вход другой

Вывод ls -l /dev передается как вход для head -20

head - команда для вывода первых строк

-20 - ключ, ограничивающий вывод первыми 20 строками

5.Изучение ключей утилиты ls
man ls

ls -R ~/Frolov          # Рекурсивный обход
ls -1 ~/Frolov          # По одному файлу на строку
ls -m ~/Frolov          # Через запятую
ls --color ~/Frolov     # Цветной вывод

6.Создание ссылок и работа с /bin

cd ~/Frolov/1 # переходим в каталог
ln group hardlink_group # ln - создание ссылок, без ключей, исходный файл group 
ln -s group symlink_group # -s - ключ для создания символической ссылки
Выводим список файлов из /bin по одному на строку, сортируем по размеру
ls -1 /bin | xargs ls -S -l 
find /bin -type f -size +15k -exec ls -lh {} \;
xargs - преобразует входной поток в аргументы командной строки

ls -S -l - подробный вывод, отсортированный по размеру

-S - сортировка по размеру (от большего к меньшему)

-l - подробный формат

/bin - каталог для поиска

-type f - искать только обычные файлы (не каталоги, ссылки и т.д.)

-size +15k - файлы размером больше 15 килобайт

-exec ls -lh {} \; - для каждого найденного файла выполнить команду

-exec:

ls -lh - человеко-читаемый подробный вывод

{} - заменяется именем найденного файла

\; - конец команды для -exec

Жесткая ссылка:

Ссылается на тот же inode (узел файловой системы)

Имеет тот же размер и содержимое

Удаление оригинала не влияет на жесткую ссылку

Проверить: ls -li покажет одинаковые inode номера

Символическая ссылка:

Это отдельный файл, содержащий путь к оригиналу

Имеет маленький размер (размер пути)

При удалении оригинала становится "битой ссылкой"

В выводе ls -l показывается как symlink_group -> group
